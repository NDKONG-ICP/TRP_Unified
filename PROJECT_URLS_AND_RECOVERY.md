#### To Recover Identity:

1. **If you have the seed phrase** (BEST option):
   ```bash
   dfx identity new ic_deploy --seed-phrase "your seed phrase here"
   ```
   **Note**: This will recreate the exact same identity if you use the original seed phrase.

2. **If you have the PEM file** (backup):
   ```bash
   dfx identity import ic_deploy_backup ~/backup/ic_identities/ic_deploy_*.pem
   ```

3. **Export current identity** (for additional backups):
   ```bash
   dfx identity export ic_deploy > ~/backup/ic_deploy_$(date +%Y%m%d).pem
   ```

#### ‚ö†Ô∏è About Seed Phrases:

**dfx does NOT store seed phrases**. When you create an identity with a seed phrase:
- The seed phrase is used **once** to generate the private key
- Only the private key (PEM file) is stored
- The seed phrase is **discarded** unless you saved it separately

**If you don't have the seed phrase**, you can still recover using the PEM file, but you cannot recreate the identity from scratch without the seed phrase.

#### Where to Store Seed Phrase:

**DO NOT store in code or git!** Store securely in:
- ‚úÖ Password manager (1Password, LastPass, Bitwarden)
- ‚úÖ Encrypted file (encrypted with GPG or similar)
- ‚úÖ Physical backup (written down, stored in safe)
- ‚úÖ Hardware wallet (if supported)

#### Identity Backup Command:
